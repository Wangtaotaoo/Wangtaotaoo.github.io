volatile

概念：java中volatile 保证修饰变量的可见性和有序性但不保证原子性，但保证 long/double等类型的原子性。

后半句的理解就是,volatile保障的原子性是他修饰的变量本身的原子性，但是不包括赋值操作的原子性：

其中 a 被修饰，但是如果b也是一个共享变量，那就无法保证原子性。即操作不具备不可分割性。

原理：
volatile的底层实现也是通过内存屏障，按照书中所写：

  • 加载屏障作用:刷新处理器缓存
  • 存储屏障作用:冲刷处理器缓存
    刷新的意思大概是获得锁之前，要把别的线程对该变量的改变最新值，立刻同步到当前的线程所操作的处理器中。
    冲刷的意思是当释放锁后，立即将自己改变的变量的值写到处理器的共享区域，保证其他线程读取到的是最新的值
  • 获取屏障:在一个读操作之后插入屏障，禁止该读操作与后面的任何读写操作之间进行重排序
  • 释放屏障：在一个写操作之前插入，禁止该写操作与前面的任何读写操作之间进行重排序
