历史

http截止到现在的时间，总共发布了四个版本
0.9（以及废弃了），1.0, 1.1 ,2.0
(其中谷歌还发明了一个版本，介于1.1与 2.0之间的一个版本，后来被2.0吸取了精髓，现在已经不用了)

1.0版本与1.1的区别

1.缓存处理 1.0中使用 if-Modified-Since来判断， http1.1加入了缓存控制策略：
（1.Entity tag, If-Unmodified-Since, If-Match, If-None-Match）
—以后再补充

2.断点续传： 1.0不支持传部分数据， 1.1支持传部分数据，（header中的range头域 - 返回码是206），这样也减轻了带宽的负担

3.增加了新的错误状态码

4.host头处理， 1.0没有传递主机名
1.1强制要求

5.长连接， 1.0是发送一次请求就断开连接（无状态的特点），1.1 可以在一个http上传送多个http请求和响应， 不需要每次请求都要重新三次握手建立连接。

附加一个面试题

  • 为什么http建立连接要三次握手？

第一次建立连接是发送一个SYN = 1的 seq = x 序列，示意服务端我要和你建立连接。


服务器发送一个 SYN = 1, ack = x+1, seq = y的序列 ， 告诉我收到了你的请求


客户端收到服务器的返回后， 再发送一个 ack = y+1, seq = x+1的序列


如果两次握手行不行？ 不行。 有种特殊情况， A 向 B 请求建立连接， 第一个请求没有收到B的回复， 这时候A会再发一个请求，这个B回复了A的请求，连接建立。 传输过后释放连接。 但是！！ A的第一个请求并没丢失，只是在某个线路上慢了， 这个时候B收到A的确认， 会误以为A发送了新的建立连接的请求，这个时候B与A建立了连接，（但是A自己都不知道这件事） B的资源就白白浪费了。
